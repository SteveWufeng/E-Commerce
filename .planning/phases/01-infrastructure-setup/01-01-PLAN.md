    Per D-01: Deploy to Cloudflare Workers via Docker + OpenNext.

    1. Install dependencies:
       ```bash
       npm install --save-dev @opennextjs/cloudflare@~1.14.0 wrangler@^4.49.1
       ```
       Version ~1.14.0 is selected because it is the latest release WITHOUT the Next.js 15 peer dependency (v1.19+ requires next >=15.5.16). This project runs Next.js 14.2.x.

    2. Create `open-next.config.ts` in project root with the following configuration:
       ```typescript
       import { defineCloudflareConfig } from "@opennextjs/cloudflare";

       export default defineCloudflareConfig({
         // Use R2 for incremental cache (included in Workers free tier)
         // Falls back to local disk cache in dev
       });
       ```
       This minimal config enables the Cloudflare adapter with defaults. The adapter handles:
       - ISR cache via Cloudflare R2 (auto-provisioned)
       - Queue worker for background tasks
       - Tag cache in D1 for revalidation

       Do NOT add r2IncrementalCache or other override imports — the default config handles this automatically in v1.14.0.

    3. Add `"type": "module"` is NOT needed at package.json level — Next.js handles ESM config files natively. Do NOT add it.

    4. Add to package.json `scripts`:
       ```json
       "cf:build": "npx @opennextjs/cloudflare build",
       "cf:deploy": "npx @opennextjs/cloudflare deploy",
       "cf:dev": "npx @opennextjs/cloudflare dev",
       "cf:typecheck": "npx tsc --noEmit"
       ```
       Keep all existing scripts. Do NOT remove `docker:build`, `docker:up`, `docker:down` — Docker remains needed for CI/CD build environment per D-01.

    5. Add to package.json `devDependencies` if not already present:
       Ensure `wrangler` is in devDependencies (added in step 1 install).
  </action>

    - **Prisma Accelerate**: Managed connection pool, paid tier.

|----------|-------------|
| Developer machine → Cloudflare Workers | Build artifacts cross trust boundary during deploy |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | Build pipeline | mitigate | OpenNext build runs in CI with locked dependency versions (package-lock.json); verify build integrity before deploy |
| T-01-02 | Information Disclosure | next.config.mjs | mitigate | No secrets in config file — all secrets via wrangler secret put; NEXT_PUBLIC_ vars only for intentional public exposure per D-09 |
| T-01-03 | Elevation of Privilege | wrangler.toml | mitigate | `nodejs_compat` flag scoped to Workers runtime; no admin-level flags enabled |

</threat_model>
